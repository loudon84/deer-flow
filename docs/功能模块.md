# DeerFlow 功能模块文档

## 一、模块概览

DeerFlow 后端由多个功能模块组成，每个模块负责特定的业务领域。本文档详细介绍各功能模块的职责、实现和使用方式。

## 二、核心模块

### 2.1 Agent 系统模块

#### 2.1.1 Lead Agent（主代理）

**位置**: `packages/harness/deerflow/agents/lead_agent/`

**职责**:
- 处理用户对话请求
- 协调工具调用和子 Agent 执行
- 管理对话上下文和状态

**核心组件**:

##### make_lead_agent 函数

**文件**: [agent.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/lead_agent/agent.py)

**功能**: 创建主 Agent 实例

**参数**:
- `config`: 运行时配置
  - `thinking_enabled`: 是否启用思考模式
  - `reasoning_effort`: 推理强度
  - `model_name`: 模型名称
  - `is_plan_mode`: 是否启用计划模式
  - `subagent_enabled`: 是否启用子 Agent
  - `agent_name`: Agent 名称

**实现细节**:
1. 解析模型配置
2. 构建中间件链
3. 加载工具集
4. 生成系统提示词
5. 创建 LangGraph Agent

##### 提示词模板

**文件**: [prompt.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/lead_agent/prompt.py)

**功能**: 生成 Agent 系统提示词

**包含内容**:
- 角色定义
- 工具使用指南
- 沙箱环境说明
- 技能列表
- 内存上下文

#### 2.1.2 Agent Factory（代理工厂）

**位置**: `packages/harness/deerflow/agents/factory.py`

**职责**: 提供纯参数化的 Agent 创建接口

**核心函数**: `create_deerflow_agent`

**特点**:
- 无配置文件依赖
- 支持自定义中间件
- 支持功能特性声明
- 支持额外中间件注入

**使用示例**:

```python
from deerflow.agents.factory import create_deerflow_agent
from deerflow.agents.features import RuntimeFeatures
from deerflow.models import create_chat_model

model = create_chat_model(name="gpt-4")
agent = create_deerflow_agent(
    model=model,
    tools=[my_tool],
    features=RuntimeFeatures(memory=True, auto_title=True),
    plan_mode=True
)
```

#### 2.1.3 Thread State（线程状态）

**位置**: `packages/harness/deerflow/agents/thread_state.py`

**职责**: 定义会话状态结构

**状态字段**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `messages` | list | 对话消息列表 |
| `sandbox` | SandboxState | 沙箱状态（sandbox_id） |
| `thread_data` | ThreadDataState | 线程数据（工作空间路径） |
| `title` | str | 会话标题 |
| `artifacts` | list[str] | 生成的文件制品路径 |
| `todos` | list | 任务列表 |
| `uploaded_files` | list[dict] | 上传的文件信息 |
| `viewed_images` | dict | 查看的图片数据 |

**Reducer 函数**:
- `merge_artifacts`: 合并并去重制品列表
- `merge_viewed_images`: 合并图片字典（支持清空）

### 2.2 中间件模块

中间件是 DeerFlow 功能扩展的核心机制，采用责任链模式处理请求和响应。

#### 2.2.1 中间件执行顺序

```
1. ThreadDataMiddleware      - 初始化线程数据
2. UploadsMiddleware          - 处理上传文件
3. SandboxMiddleware          - 初始化沙箱环境
4. DanglingToolCallMiddleware - 修复悬空工具调用
5. GuardrailMiddleware        - 护栏检查
6. ToolErrorHandlingMiddleware - 工具错误处理
7. SummarizationMiddleware    - 对话摘要
8. TodoMiddleware             - 任务列表管理
9. TitleMiddleware            - 自动标题生成
10. MemoryMiddleware          - 对话记忆管理
11. ViewImageMiddleware       - 图片查看处理
12. SubagentLimitMiddleware   - 子 Agent 并发限制
13. LoopDetectionMiddleware   - 循环检测与中断
14. ClarificationMiddleware   - 澄清请求处理
```

#### 2.2.2 关键中间件详解

##### MemoryMiddleware

**文件**: [memory_middleware.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/middlewares/memory_middleware.py)

**功能**: 管理对话记忆，支持长期记忆存储

**实现机制**:
1. 过滤消息（只保留用户输入和最终响应）
2. 检测用户纠正信号
3. 将对话加入内存更新队列
4. 异步更新内存存储

**消息过滤规则**:
- 保留 HumanMessage（移除上传文件标签）
- 保留 AIMessage（无 tool_calls）
- 移除 ToolMessage
- 移除 AI 消息中的工具调用

**纠正检测模式**:
- 英文: "that's wrong", "you misunderstood", "try again", "redo"
- 中文: "不对", "你理解错了", "重试", "重新来"

##### LoopDetectionMiddleware

**文件**: [loop_detection_middleware.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/middlewares/loop_detection_middleware.py)

**功能**: 检测并中断重复工具调用循环

**检测策略**:
1. 哈希工具调用（name + args）
2. 在滑动窗口中跟踪调用哈希
3. 相同哈希出现 3 次：注入警告消息
4. 相同哈希出现 5 次：强制移除工具调用

**配置参数**:
- `warn_threshold`: 警告阈值（默认 3）
- `hard_limit`: 强制中断阈值（默认 5）
- `window_size`: 滑动窗口大小（默认 20）
- `max_tracked_threads`: 最大跟踪线程数（默认 100）

##### TitleMiddleware

**文件**: [title_middleware.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/middlewares/title_middleware.py)

**功能**: 自动生成会话标题

**触发条件**:
- 首次对话完成后
- 标题为空时

**实现方式**:
- 使用 LLM 生成简短标题
- 限制标题长度
- 异步更新标题

##### TodoMiddleware

**文件**: [todo_middleware.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/middlewares/todo_middleware.py)

**功能**: 任务列表管理，支持计划模式

**提供的工具**: `write_todos`

**任务状态**:
- `pending`: 待处理
- `in_progress`: 进行中
- `completed`: 已完成

**使用场景**:
- 复杂多步骤任务（3+ 步骤）
- 需要系统规划的任务
- 用户明确要求任务列表

### 7.1 DeerFlow Client

**文件**: [client.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/client.py)

**功能**: 嵌入式 Python 客户端

**核心方法**:

| 方法 | 功能 |
|------|------|
| `chat()` | 发送消息并返回响应 |
| `stream()` | 流式发送消息 |
| `list_models()` | 列出可用模型 |
| `list_skills()` | 列出可用技能 |
| `get_memory()` | 获取内存数据 |
| `update_memory()` | 更新内存数据 |

**使用示例**:

```python
from deerflow.client import DeerFlowClient

client = DeerFlowClient(
    model_name="gpt-4",
    thinking_enabled=True
)

response = client.chat("Hello!")
print(response)

for event in client.stream("Tell me a story"):
    print(event.type, event.data)
```
