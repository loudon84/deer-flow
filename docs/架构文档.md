# DeerFlow 后端架构文档

## 一、系统概述

DeerFlow 是一个基于 LangGraph 的 AI Agent 系统，提供沙箱执行能力和多渠道集成支持。系统采用模块化设计，支持多种大语言模型、工具扩展和即时通讯平台集成。

### 核心特性

- **LangGraph Agent 框架**：基于 LangGraph 构建的状态机驱动的 Agent 系统
- **沙箱执行环境**：支持本地和远程沙箱，安全执行代码和命令
- **多模型支持**：支持 OpenAI、Claude、DeepSeek、Minimax 等多种 LLM
- **中间件架构**：通过中间件链实现功能扩展（内存、标题生成、循环检测等）
- **多渠道集成**：支持飞书、Slack、Telegram 等即时通讯平台
- **MCP 协议支持**：集成 Model Context Protocol，扩展工具能力

## 三、核心组件

### 3.1 Agent 系统

Agent 系统是 DeerFlow 的核心，基于 LangGraph 构建，采用中间件架构实现功能扩展。

#### 3.1.1 Lead Agent

主 Agent 负责处理用户请求，协调工具调用和子 Agent 执行。

**关键文件**：
- [agent.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/lead_agent/agent.py) - Agent 工厂函数
- [factory.py](file:///d:/git_ai/deer-flow/backend/packages/harness/deerflow/agents/factory.py) - 通用 Agent 创建函数

**核心功能**：
- 模型解析与创建
- 中间件链构建
- 工具加载与注入
- 系统提示词生成

#### 3.1.2 Thread State

线程状态定义，管理会话上下文和元数据。

**状态字段**：
- `messages`: 对话消息列表
- `sandbox`: 沙箱状态
- `thread_data`: 线程数据（工作空间路径等）
- `title`: 会话标题
- `artifacts`: 生成的文件制品
- `todos`: 任务列表
- `uploaded_files`: 上传的文件
- `viewed_images`: 查看的图片

#### 3.1.3 Checkpointer

检查点持久化，支持多种后端存储。

**支持的后端**：
- Memory: 内存存储（默认）
- SQLite: SQLite 数据库
- Postgres: PostgreSQL 数据库

### 3.2 中间件系统

中间件是 DeerFlow 功能扩展的核心机制，采用责任链模式。

#### 中间件执行顺序

```
ThreadDataMiddleware → UploadsMiddleware → SandboxMiddleware
    → DanglingToolCallMiddleware → GuardrailMiddleware
    → ToolErrorHandlingMiddleware → SummarizationMiddleware
    → TodoMiddleware → TitleMiddleware → MemoryMiddleware
    → ViewImageMiddleware → SubagentLimitMiddleware
    → LoopDetectionMiddleware → ClarificationMiddleware
```

## 四、数据流

### 4.1 请求处理流程

```
用户请求
    ↓
Gateway (FastAPI)
    ↓
ThreadRuns Router
    ↓
RunManager (创建运行)
    ↓
Lead Agent (LangGraph)
    ↓
Middleware Chain
    ↓
Tool Execution / Model Call
    ↓
Response Stream
    ↓
Gateway (SSE)
    ↓
用户
```

### 4.2 消息流转

```
HumanMessage
    ↓
ThreadDataMiddleware (初始化线程数据)
    ↓
UploadsMiddleware (处理上传文件)
    ↓
SandboxMiddleware (初始化沙箱)
    ↓
[其他中间件处理]
    ↓
Model Call (LLM 推理)
    ↓
Tool Calls (工具调用)
    ↓
[工具执行]
    ↓
ToolMessage
    ↓
[循环直到完成]
    ↓
AIMessage (最终响应)
    ↓
MemoryMiddleware (队列内存更新)
    ↓
TitleMiddleware (生成标题)
    ↓
返回用户
```

## 六、扩展机制

### 6.1 添加新中间件

1. 继承 `AgentMiddleware` 类
2. 实现 `after_model` 或其他钩子方法
3. 在 `_build_middlewares` 中注册

### 6.2 添加新工具

1. 创建工具类（继承 `BaseTool`）
2. 在配置文件中注册
3. 通过 `get_available_tools` 加载

### 6.3 添加新渠道

1. 继承 `Channel` 基类
2. 实现消息解析和发送方法
3. 在 `_CHANNEL_REGISTRY` 中注册
